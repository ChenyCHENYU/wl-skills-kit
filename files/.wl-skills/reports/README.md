# Reports 目录

本目录存放 AI 生成的报告类文件。**所有写入均为追加模式，不覆盖历史。**

---

## 文件分类

### A. 系统数据文件

由生成类 Skill 写入，由同步类 Skill 读取。

| 文件                               | 写入方             | 读取方          | 说明                   |
| ---------------------------------- | ------------------ | --------------- | ---------------------- |
| `SYS_MENU_INFO.md`                 | page-codegen       | menu-sync       | 页面生成时追加菜单条目 |
| `SYS_DICT_INFO.md`                 | dict-collect       | dict-sync       | 字典数据汇总           |
| `SYS_PERMISSION_INFO.md` [PLANNED] | permission-collect | permission-sync | 权限数据汇总（可选基线，当前 permission-sync 不强依赖） |

**写入规则**：每条数据带生成时间戳；新条目追加到末尾；不删除历史。

### B. 审计偏差报告

| 文件              | 写入方           | 读取方                 | 说明                    |
| ----------------- | ---------------- | ---------------------- | ----------------------- |
| `规范审查报告.md` | convention-audit | 人工 review → code-fix | 不合规项清单 + 整改建议 |

**写入规则**：每次审计追加**新章节在顶部**（最新在最上），时间戳分节，历史自然下沉。

### C. 提取建议报告

| 文件              | 写入方           | 读取方                         | 说明                         |
| ----------------- | ---------------- | ------------------------------ | ---------------------------- |
| `组件提取建议.md` | convention-audit | 人工 review → template-extract | 跨页面复用但未封装的模式清单 |

**写入规则**：同 B。

---

## 为什么追加不覆盖

1. **历史可追溯**：通过 git diff 可看到团队每次审计的变化趋势
2. **多次执行幂等**：开发者可重复触发，不丢失数据
3. **多人并行**：多人协作时不会互相覆盖
4. **数据闭环**：菜单/字典/权限同步类 Skill 依赖累积数据

## 安装与升级行为

`bin/wl-skills.js` 对本目录的特殊处理：

| 命令     | 行为                                                             |
| -------- | ---------------------------------------------------------------- |
| `init`   | 文件不存在时创建空模板；**已存在则跳过**（避免覆盖团队累积数据） |
| `update` | 同 `init`：已存在的文件**不覆盖**                                |
| `clean`  | 默认删除（视为 AI 辅助文件）；可加 `--keep-reports` 保留         |

## Git 管理建议

- ✅ `SYS_MENU_INFO.md`、`SYS_DICT_INFO.md`、`SYS_PERMISSION_INFO.md` 提交到仓库（团队共享数据）
- ⚠️ `规范审查报告.md`、`组件提取建议.md` 视团队偏好选择：
  - 提交：可见整改进度，PR review 时直观
  - 不提交：避免污染 git 历史，加入 `.gitignore`
